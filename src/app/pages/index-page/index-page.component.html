Credit Card Form

Create a form for collecting credit card information, under the New Credit Card Page.

Requirements

  • The form should have the following fields:
    Nickname
    Required, the card's nickname, at least 1 character.
    Cardholder's First Name
    Required, at least 1 character.
    Cardholder's Last Name
    Required, at least 1 character.
    Billing Zip
    Required, 5 digits long.
    Card Number
    Required, 16 digits long.
  • When the form is invalid, the save button should be disabled.
  • When the form is valid, the save button should be enabled.
  • When the enabled save button is clicked, call the '#save' method on the CreditCardService with the form contents
    • While the save is ongoing, disable the button.
    • If the save is successful, navigate back to the credit cards route.
    • If the save fails, show the error message and re-enable the save button.

"New Credit Card" in Invoices

Hook the form you just created up to 'credit card select' field on the Invoice Payment Page.

Requirements

  • Use Angular MatDialog to launch the modal.
  • The requirements of form validation have not changed.
  • If the user clicks cancel in the modal, the modal should close and the card select field should be blank.
  • If the user clicks save in the modal, call the '#save' method on the CreditCardServices with the form contents.
    • While the save is ongoing, all modal buttons are disabled.
    • If the save is successful, hide the modal, and ensure the new card is selected in the form.
    • If the save fails, don't hide the model, and show the error message in the form.
